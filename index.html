Usage

Configure

To use the Android Server-Side Tagging (SST) SDK it must be configured by calling the Sst.configure method.
This method takes a Config object and a ContextProvider as parameters. The Config object is used to configure the SDK with the necessary information to send events to the server. The ContextProvider is used to provide the SDK with the android context used to extract the necessary information to enrich the events.

val config = Config("client_name")
Sst.configure(sstConfig) { this }

Track Event

To track an event, call the Sst.trackEvent method, passing it an Event object as a parameter. The Event and included model information is encoded according to the SST Protocol before being sent to SST.

val event = Event("launch")
Sst.trackEvent(event)

To include additional data for the event, populate the data property.

val event = Event("screen_view", mapOf("screen_name" to "Home"))
Sst.trackEvent(event)

To include additional query parameters for the event, populate the parameters property.

val event = Event("screen_view", parameters = mapOf("ensDisableTracking" to "user"))

Persistent Data Layer

The SDK includes a simple key-value store, called the DataLayer, that can be used to store data that is persisted to disk and included in every request sent to SST.

Sst.dataLayer.get<Int>("launch_count")?.let {
    Sst.dataLayer.add("launch_count", it + 1)
} ?: Sst.dataLayer.add("launch_count", 1)

CHEQ UUID

The CHEQ UUID is automatically generated and populated by the SST servers. This unique identifier is persisted to disk by the SDK, ensuring it is available across multiple sessions and app launches. The UUID is a critical component in identifying and tracking users or sessions, and it is included on every request to SST.

// retrieve CHEQ UUID
val uuid = Sst.getCheqUuid()

// clear the stored CHEQ UUID
Sst.clearCheqUuid()
